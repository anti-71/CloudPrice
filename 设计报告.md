## （一）需求分析

### 1.1 项目背景

消费者价格指数（CPI）是衡量通胀水平的重要宏观经济指标，但传统 CPI 依赖人工调研与月度统计更新，存在**时效性差、颗粒度粗、覆盖不全**等缺陷。随着电商平台数据大规模生成，构建高频（按日）价格指数成为可能。本项目利用阿里云 OSS 对象存储与 ClickHouse 列式数据库，基于 Python 实现一套自动化、可复用的电商价格指数计算体系。

### 1.2 功能需求

| 需求 | 说明 |
|------|------|
| **数据管理** | 上传学习通真实数据集（category.csv / products.csv / 每日 price.csv）至 OSS |
| **数据清洗** | 过滤异常数据（负价格、空名称、价格暴涨等），异常过滤率 ≥ 95% |
| **价格指数计算** | 按分类权重（CPI 官方权重）计算加权价格指数 |
| **可视化输出** | 通过 matplotlib 生成日度指数趋势折线图 |
| **CI/CD** | GitHub Actions 自动触发格式校验、单元测试、集成测试、部署 |

### 1.3 数据规模

- 真实数据集：**1095 个 CSV 文件，共 1.5GB，2956 万行**
- 模拟数据集：**770 万行**（2000 SKU × 3 平台 × 1096 天）

### 5.2 数据清洗 (src/compute/data_cleaner.py)

```python
def clean(self, df: pd.DataFrame) -> pd.DataFrame:
    df = df[df["price"] > 0]                    # 过滤负价格
    df = df[df["sales"] >= 0]                   # 过滤负销量
    df = df[df["product_name"].notna()]         # 过滤空名称
    df = df[df["price"] <= median * 10]         # 过滤价格暴涨
    df = df.drop_duplicates(subset=["date", "sku_id", "platform", "shop_id"])
    return df
```

### 5.3 指数计算 (src/compute/index_computer.py)

```python
def overall_index(self, ci, cat):
    merged = ci.merge(cat[["date", "category", "total_sales"]], ...)
    merged["weight"] = merged.groupby("date")["total_sales"].transform(
        lambda x: x / x.sum())
    oi = merged.groupby("date").apply(
        lambda g: (g["index"] * g["weight"]).sum())
    return oi  # 加权类目指数
```

### 5.4 真实 CPI 数据处理 (scripts/process_real_data.py)

```python
# 用 CPI 官方权重代替销量
df["index"] = df["price"] / df["base_price"] * 100
df["weight"] = df["product_id"].map(prod_w)  # 商品权重
cat_w = leaf.set_index("category_id")["weight"]  # 类目权重

# 类目加权 → 整体加权
num = df.groupby("category_id").apply(
    lambda x: (x["index"] * x["weight"]).sum())
den = df.groupby("category_id")["weight"].sum()
ov = (num/den * cat_w).sum() / cat_w.sum()  # 最终 CPI 指数
```

## （六）核心 SQL 代码逻辑

### 6.1 ClickHouse 建表

```sql
CREATE TABLE IF NOT EXISTS ods_raw_prices (
    date Date, sku_id String, product_name String,
    category_l1 String, category_l2 String, brand String,
    price Float64, sales Int64
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(date)
ORDER BY (date, sku_id);
```

### 6.2 异常处理（ClickHouse SQL）

```sql
-- 清洗：过滤 null、负价格、价格暴涨
INSERT INTO dwd_clean_prices
SELECT date, sku_id, product_name, category_l1, category_l2, brand, price, sales
FROM ods_raw_prices
WHERE price > 0
  AND sales >= 0
  AND product_name IS NOT NULL AND product_name != ''
  AND price <= (SELECT median(price) * 10 FROM ods_raw_prices);
```

### 6.3 汇总查询

```sql
-- SKU 日汇总
INSERT INTO dws_sku_daily_summary
SELECT date, sku_id, product_name, category_l1, category_l2, brand,
       avg(price) AS avg_price, sum(sales) AS total_sales, count() AS record_count
FROM dwd_clean_prices
GROUP BY date, sku_id, product_name, category_l1, category_l2, brand;

-- 全网日汇总
INSERT INTO dws_overall_daily_summary
SELECT date, avg(price), sum(sales), count(DISTINCT sku_id), count()
FROM dwd_clean_prices
GROUP BY date;
```

### 6.4 折线图数据查询

```sql
SELECT date, index_value
FROM cpi_index_daily
ORDER BY date;
```
