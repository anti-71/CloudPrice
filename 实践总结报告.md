## 一、需求分析

### 1.1 项目背景

消费者价格指数（CPI）是衡量通胀水平的宏观指标，传统 CPI 依赖人工月度统计，存在时效性差、颗粒度粗等缺陷。本项目利用电商平台每日产生的海量价格数据，构建高频（按日）价格指数，实现对市场更灵敏的监测。

### 1.2 核心需求

- 接入课程下发的真实 CPI 数据集（1095 个 CSV，2956 万行，1.5GB）
- 对原始数据进行异常清洗（空值、负价、价格暴涨）
- 按照 CPI 官方权重计算加权价格指数
- 通过 ClickHouse SQL 实现高性能聚合计算
- 生成日度价格指数折线图
- 配置 GitHub Actions 实现 CI/CD 自动化
- 所有密钥通过 GitHub Secrets 传入，不提交到仓库

## 四、SQL 算法及关键语句分析

### 4.1 异常处理（ClickHouse SQL）

```sql
-- 清洗规则：过滤负价格、负销量、空名称、价格暴涨
INSERT INTO dwd_clean_prices
SELECT date, sku_id, product_name, category_l1, category_l2, brand, price, sales
FROM ods_raw_prices
WHERE price > 0                    -- 排除负价格
  AND sales >= 0                   -- 排除负销量
  AND product_name != ''           -- 排除空名称
  AND price <= (                   -- 排除价格暴涨（超过中位数10倍）
      SELECT median(price) * 10 FROM ods_raw_prices
  );
```

**分析**：`median()` 比 `avg()` 更适合做异常检测基准，因为平均数会被极端值拉偏而中位数不受影响。乘以 10 的阈值确保正常的价格波动（如促销）不被误杀。

### 4.2 权重计算（真实 CPI 数据）

```python
# 商品级指数 = 当日价格 / 基期价格 × 100
df["index"] = df["price"] / df["base_price"] * 100

# 类目加权（类目内按商品权重）
num = df.groupby("category_id").apply(
    lambda x: (x["index"] * x["weight"]).sum())
den = df.groupby("category_id")["weight"].sum()
cat_index = num / den

# 整体 CPI = 各类目按官方权重加权
overall = (cat_index * cat_weight).sum() / cat_weight.sum()
```

**分析**：采用拉氏指数思路，以基期价格固定权数，逐日计算价格变动。与教材中的阿根廷 CPI 算法一致（参考《Online and official price indexes: Measuring Argentina's inflation》3.1 节）。

### 5.2 ClickHouse 客户端（src/storage/clickhouse_client.py）

```python
class ClickHouseClient:
    def __init__(self, config):
        self.base_url = f"http://{config.clickhouse_host}:{config.clickhouse_port}"
        self._auth = (config.clickhouse_user, config.clickhouse_password)

    def query(self, sql: str) -> str:
        resp = requests.get(self.base_url, params={"query": sql}, auth=self._auth)
        return resp.text

    def insert_tsv(self, table: str, tsv_data: str):
        resp = requests.post(self.base_url,
            params={"query": f"INSERT INTO {table} FORMAT TabSeparated"},
            data=tsv_data.encode("utf-8"), auth=self._auth)
```

**实现功能**：封装 ClickHouse HTTP 接口，支持 GET 查询和 TSV 批量插入。避免 `INSERT VALUES` 的 URL 长度 404 限制。

## 八、实验心得与课程反馈

### 8.1 技术收获

1. **pandas 是大数据处理的标配**：从 dataclass 迁移到 DataFrame 后，770 万行数据处理从 25 分钟降到 2 分钟，体验了向量化运算的威力。

2. **ClickHouse 不只是"另一个数据库"**：列式存储 + OLAP 设计使得 2956 万行聚合查询从 MySQL 的 6 分钟降到 2 秒，深刻理解了 OLTP 和 OLAP 的本质区别。

3. **云服务的成本意识**：从 4核16GB 到 2核8GB 的降级决策，月费从 2400 元降到 600 元，学会了"按需选型"。

4. **CI/CD 不是摆设**：GitHub Actions 自动化流程让每次 push 都能自动验证，4 个 Stage 的分离设计保证了核心和非核心步骤互不阻塞。

### 8.2 踩过的坑

- ClickHouse 白名单不放开导致 120 秒连接超时，整整排查了 2 个小时
- `INSERT VALUES` 批量插入遇到 URL 长度 404 限制，最终改用 TSV 格式
- 真实数据集 `categories.csv` 和 `daily_price/*.csv` 是 GBK 编码，pandas 默认 UTF-8 报错
- 初版用销量加权导致指数飙到 1534，发现数据根本没有销量字段

### 8.3 课程反馈

本课程设计将云存储、OLAP 数据库、Python 自动化、CI/CD 串联成一个完整的工业化流程，覆盖面广且每个环节都有实际操作，收获远超单一知识点的学习。建议后续可以考虑加入实时流处理（如 Flink/Flink CDC），让学生体验实时价格指数的计算。
